# Sparkle Benchmark Suite

Compares Sparkle JIT simulation against Verilator on LiteX PicoRV32 SoC designs.

## Quick Start

```bash
cd bench
make              # Run all benchmarks (LiteX 1-core + 8-core parallel)
make litex        # LiteX 1-core only
make multicore    # 8-core parallel only
make rv32         # RV32I SoC (Sparkle-native) only
```

## Prerequisites

- **verilator** — `apt install verilator` or `nix-shell`
- **g++** with C++20 (`-std=c++20` for `std::barrier`)
- **lake** — Lean 4 build tool
- **python3**
- PicoRV32 is auto-fetched from GitHub if not cached

## Benchmarks

### 1. LiteX 1-Core (`make litex`)

Single PicoRV32 core with UART, timer, SRAM, main RAM.
1730-line real-world SoC generated by LiteX/Migen.

```
==========================================
  LiteX 1-Core Benchmark (10000000 cycles)
==========================================
Verilator:    10.34 M cyc/s
Sparkle JIT:  11.65 M cyc/s    ← 1.13x faster
==========================================
```

### 2. Multi-Core Parallel (`make multicore`)

8 PicoRV32 cores connected via shared bus.
Sparkle uses CDC lock-free multi-thread runner.

```
==========================================
  Multi-Core Benchmark (10000000 cycles)
==========================================
JIT 1-core:          11.57 M cyc/s
JIT 8-core seq:       1.46 M per-core cyc/s
JIT 8-core parallel:  5.07 M per-core cyc/s    ← 3.47x speedup
Verilator 8-core:     1.06 M cyc/s
==========================================
```

### 3. RV32I SoC (`make rv32`)

Sparkle-native RV32I SoC (smaller, simpler design).

## How Multi-Thread Parallel Works

1. Each core gets its own `sim_core` C++ class instance
2. N threads run `sim_core::evalTick()` in parallel
3. Shared bus data exchanged via `std::atomic` after each batch
4. `std::barrier` synchronizes threads between batches
5. Batch size controls sync frequency (10K cycles = sweet spot)

```
Thread 0: [eval 10K cycles] → barrier → [exchange bus] → barrier → repeat
Thread 1: [eval 10K cycles] → barrier → [exchange bus] → barrier → repeat
...
Thread 7: [eval 10K cycles] → barrier → [exchange bus] → barrier → repeat
```


- `dev/rv32-bench` — RV32I SoC tracking
- `dev/litex-bench` — LiteX 1-core tracking
- `dev/multicore-bench` — 8-core parallel tracking
